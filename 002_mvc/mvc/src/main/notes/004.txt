헤더

공통헤더, 요청,응답메시지 전용헤더, 커스텀

{}
1로 시작하는 코드는 특수한 경우에만 쓴다. (설명X) 
2부터 시작하는 코드>
 - 성공적으로 수행중 
 - 200ok: 요청을 했을 때 성공했을 때(가장 많이 쓰이는 상태코드이다. ) 
 - 201create: -  요청을 보냈을 때 리소스가 생성이되었을 때 
                 - 대부분의 post 요청에서 쓰인다. 
 	     -  권장된다. 
	ex) 게시글을 쓸 때등(post요청)
 - 202 Accepted
 	- 전학년 생기부를 쓸 때 ( 시간이 걸림) , 통계를 낼 때(요청을 보내고 바로 답이 올 수 없다. ) 
	- 요청을 주고 진행결과를 보여줄 수 있는 페이지로 이동한다. 
	- 비동기 작업이 일어날 때 사용되는 코드 
- 204 NoContent
	- 성공을 해서 응답을 해주면 될 때 바디가 없을 때
	- 말 그대로 작업이 성공하였고 바디에 포함된 정보는 없다는 의미
	DELETE 메소드 요청에 대한 응답으로 적절 (성공적으로 삭제 작업이 수행되었으며 삭제 결과로 딱히 전달할 데이터가 없으므로 바디에 정보는 생략)
- 206 Partial Content
	-대용량 파일을 다운로드 할 때 일부 내용을 줄 수 있다. 이것을 파셜 컨텐츠로한다. 


{ 300 }
- 301 Moved Permanently
요청한 리소스 URL이 영구적(Permanently)으로 다른 URL로 교체됨
url 을 개편해 완전히 새로운 사이트로 이동할 때 전 사이트에서는 바뀐 사이트로 연결할 때 쓰인다. 
Location 헤더를 이용하여 교체된 URL 정보를 전달하는 것은 필수이다. 

- 302 Found
	- 임시대피소 
	- 북마크(url)를 수정할 필요가 없다. 
- 303 See Other
	- 해당 주소와 직접적 혹은 간접적인 정보를 포함한 다른 URL 리소스로의 이동 처리를 위해서 사용됨
	- 응답을 찾을 수 있는 곳에 대한 힌트를 준다. 
- 304 Not Modified
	- 요청한 문서가 가장 최신이라 브라우저에 저장된 캐시를 사용해도 된다고 응답해준다. 
	- 캐시 관련 
{4XX (실패 : 클라이언트 오류)}
- 400 Bad Request
	- 사용자가 요청을 잘못한 경우이다. (오타 , 필수 정보,이메일을 잘못 보낼 때) 
-  401 Unauthorized
	- 인증이 안된 것 (만약 교복을 입고 있으면 학생인 줄 알지만, 수상한 사람은 교문을 통과할 수 없다.) 
	- 로그인이 되어야만 볼 수 있는 글을 로그인 안된 사람이 보려고 하면 나오는 에러이다. 
	- 일반 유저가 관리자 페이지를 보려고 접속할 때 나온다. 
- 404 Not Found
	- 해당 요청에 대한 모든 리소스(주로 웹 페이지)가 존재하지 않음
	- 꼭 웹 페이지가 존재하지 않을 때만 사용해야 하는 코드는 아니며 모든 자료(ex: json, xml)를 요청했을 때 해당 자료가 없으면 404 코드 반환 가능

- 405 Method Not Allowed
	- 허용하지 않는 HTTP 메소드를 이용하여 요청 (GET 요청만 받는 핸들러에 POST 요청 보내기 상황)
	- 원래 이 path에는 post 를 보내기로 했는데, get을 보내면 에러가 난다. 
- 406 Not Acceptable
	- 내용 협상(Content Negotiation)에 실패하여 반환할 적당한 리소스가 없음
	- 잘 사용되지 않음 (오류 코드를 보는 것보다는 나으니 차라리 그냥 대안값으로 응답해버림)
	- 완고한 사람. html을 달라고 했을 때 json밖에 없을 때 

- 414 URI Too Long
	- 요청 메시지에 포함된 주소(Path)가 너무 길 경우 발생 (보통 서버는 8KB까지 주소 정보를 받을 수 있도록 설정)
	-파일을  body를 통해 보내야 하는데 주소가 너무 길 경우에 발생하는 에러이다. 

{ 5XX (실패 : 서버 오류) }
- 진짜 개발자가 실패한 경우 
- 500 Internal Server Error
	- 서버 내부에서 발생한 오류( 예외 발생등) 으로 인해 요청처리 불가 
	- 시스템이 죽은 경우 
- 503 Service Unavailable
	- 티켓팅과 같은 경우 일시적으로 서비스를 들어갈 수 없을 때 
	- 서버증원을 해야 했다.
